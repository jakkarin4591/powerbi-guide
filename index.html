📊 แผนภูมิแท่งและแผนภูมิคอลัมน์ (Bar chart & Column chart)
Bar chart & Column chart เป็นวิชวลที่นิยมใช้งานกัน ซึ่งใช้ในการเปรียบเทียบปริมาณข้อมูลแต่ละกลุ่ม
• Bar chart — แกน X เป็นค่ากลุ่ม, แกน Y เป็นค่าตัวเลข (แท่งแนวนอน)
• Column chart — แกน X เป็นค่าตัวเลข, แกน Y เป็นค่ากลุ่ม (แท่งแนวตั้ง)

ทั้งสองชนิดนี้ เราจะเลือกใช้เมื่อข้อมูลมีปริมาณไม่มาก โดย Bar chart นิยมใช้เปรียบเทียบข้อมูลต่างชนิดกันที่เวลาเดียวกัน ส่วน Column chart นิยมใช้ในการเปรียบเทียบข้อมูลชนิดเดียวกันที่เวลาต่างกัน

Bar Chart (แนวนอน)
Stacked / Clustered / 100% Stacked
แสดงข้อมูลแบบแท่งแนวนอน เหมาะเมื่อชื่อหมวดหมู่มีความยาว หรือต้องการเรียงลำดับค่าจากมากไปน้อยให้ชัดเจน
เมื่อไหร่ควรใช้: เปรียบเทียบยอดขายหลายสาขา, คะแนนแต่ละวิชา, อันดับค่าครองชีพ
✅ ข้อดีอ่านง่าย ชื่อยาวไม่ตัด มองเห็น Ranking ชัด
⚠️ ข้อเสียมีหมวดหมู่มากเกิน 15+ ดูยาก
📈
Column Chart (แนวตั้ง)
Stacked / Clustered / 100% Stacked
แสดงข้อมูลแบบแท่งแนวตั้ง นิยมใช้เปรียบเทียบข้อมูลชนิดเดียวกันที่เวลาต่างกัน เช่น รายเดือน รายไตรมาส
เมื่อไหร่ควรใช้: ยอดขายรายเดือน, งบประมาณรายปี, จำนวนผู้ใช้งานรายไตรมาส
✅ ข้อดีมาตรฐาน เข้าใจง่าย รองรับ Stacked / Clustered
⚠️ ข้อเสียชื่อหมวดยาวทับกัน ต้องหมุน Label
📈 แผนภูมิเส้นและแผนภูมิพื้นที่ (Line chart & Area chart)
Line chart & Area chart ทั้งสองรูปแบบใช้แสดงความเปลี่ยนแปลงของปริมาณข้อมูล โดยแกนแนวนอนแสดงค่าเป็นกลุ่มข้อมูล และแกนแนวตั้งแสดงค่าเป็นค่าตัวเลข สามารถแยกย่อยได้ 3 รูปแบบ:
1. Line chart — เหมาะสำหรับแสดงแนวโน้มโดยรวมของชุดข้อมูลที่มีจุดข้อมูลจำนวนมาก
2. Area chart — เน้นให้เห็นถึงการเปลี่ยนแปลงปริมาณข้อมูลในแต่ละช่วงเวลาด้วยพื้นที่สีใต้เส้น
3. Stacked area chart — เหมาะสำหรับข้อมูลที่ต้องแสดงแนวโน้มหรือเน้นขนาดการเปลี่ยนแปลงในช่วงเวลาใดเวลาหนึ่ง
📉
Line Chart (แผนภูมิเส้น)
แสดงแนวโน้มของข้อมูลตามเวลา
แสดงแนวโน้มของข้อมูลตามลำดับเวลา เหมาะกับชุดข้อมูลที่มีจุดข้อมูลจำนวนมาก รองรับหลาย Series พร้อมกัน
เมื่อไหร่ควรใช้: ยอดขายรายวัน/เดือน/ปี, อุณหภูมิรายวัน, จำนวนผู้ป่วยตามเวลา
✅ ข้อดีเห็น Trend ชัดเจน รองรับข้อมูลจำนวนมาก
⚠️ ข้อเสียไม่เหมาะกับข้อมูลที่ไม่ต่อเนื่อง
⛰️
Area Chart (แผนภูมิพื้นที่)
เน้นปริมาณและแนวโน้มพร้อมกัน
คล้าย Line chart แต่เติมสีใต้เส้น ทำให้เห็นทั้งแนวโน้มและปริมาณ ช่วยในการดูและเปรียบเทียบแนวโน้มปริมาณข้อมูลเพิ่มเติมได้
เมื่อไหร่ควรใช้: ปริมาณน้ำฝน, ปริมาณการใช้พลังงาน, จำนวนผู้ใช้งานสะสม
✅ ข้อดีเห็นทั้ง Trend และปริมาณในภาพเดียว
⚠️ ข้อเสียซ้อนกันมากทำให้อ่านยาก ควรใช้ 2-3 Series
🔀 แผนภูมิเส้นและแผนภูมิคอลัมน์ (Line and Column chart)
Line and column chart เป็นวิชวลผสมใน Power BI โดยผสมระหว่างแผนภูมิเส้นและแผนภูมิคอลัมน์ ซึ่งช่วยให้เปรียบเทียบข้อมูลได้เร็วขึ้น โดยแผนภูมิจะมีแกน Y หนึ่งหรือสองแกน (วางอยู่ซ้ายและขวา) เราจะเลือกใช้เมื่อ:
• ต้องการแสดงความสัมพันธ์ระหว่างสองหน่วยวัดในการแสดงผล
• เมื่อข้อมูลเรามีแผนภูมิเส้นและแผนภูมิคอลัมน์ที่ใช้แกน X เดียวกัน
🔀
Line and stacked column chart
แผนภูมิเส้นและแผนภูมิคอลัมน์แบบเรียงซ้อน
ผสมแผนภูมิคอลัมน์แบบเรียงซ้อนกับเส้น เหมาะกับการแสดงข้อมูล 2 มิติพร้อมกัน เช่น ปริมาณฝนรายเดือน (คอลัมน์) และอุณหภูมิเฉลี่ย (เส้น)
เมื่อไหร่ควรใช้: ปริมาณน้ำฝนและอุณหภูมิ, ยอดขายและอัตรากำไร
✅ ข้อดีเปรียบเทียบ 2 หน่วยวัดได้ในกราฟเดียว
⚠️ ข้อเสียซับซ้อน ผู้อ่านต้องทำความเข้าใจ 2 แกน Y
🔀
Line and clustered column chart
แผนภูมิเส้นและแผนภูมิกลุ่มคอลัมน์
ผสมแผนภูมิคอลัมน์แบบกลุ่มกับเส้น แสดงข้อมูลได้ชัดเจนขึ้นเมื่อมีข้อมูลหลาย Category
เมื่อไหร่ควรใช้: เปรียบเทียบยอดขายหลายกลุ่มพร้อมกับแสดงแนวโน้มรวม
✅ ข้อดีเห็นรายละเอียดแต่ละกลุ่มและ Trend พร้อมกัน
⚠️ ข้อเสียข้อมูลหลาย Series ดูรกได้
🎀 แผนภูมิริบบอน (Ribbon chart)
Ribbon chart (แผนภูมิริบบอน) เป็นแผนภูมิภาพที่ใช้ค้นหาได้อย่างรวดเร็วว่าข้อมูลกลุ่มใดมีอันดับสูงสุด (มีค่ามากที่สุด) และกลุ่มใดมีอันดับต่ำสุด (มีค่าน้อยที่สุด)

แผนภูมิริบบอนเหมาะสำหรับการแสดงการเปลี่ยนแปลงอันดับข้อมูลต่างๆ ในช่วงเวลา (แนวแกน X) โดยจะกำหนดค่าอันดับสูงสุดให้แสดงอยู่ด้านบนสุดของช่วงเวลาเสมอ
🎀
Ribbon Chart
ติดตามการเปลี่ยนแปลงอันดับในแต่ละช่วงเวลา
Ribbon Chart แสดงการเปลี่ยนแปลงอันดับของแต่ละกลุ่มข้อมูลในแต่ละช่วงเวลา ค่าที่มากที่สุดจะแสดงอยู่บนสุดเสมอ ทำให้เห็นว่ากลุ่มไหนขึ้นหรือลงอันดับ
เมื่อไหร่ควรใช้: คะแนนสอบแต่ละวิชาแต่ละเดือน, อันดับค่าครองชีพ, ข้อมูลรายจ่ายแต่ละแผนกตามเวลา
✅ ข้อดีเห็น Ranking เปลี่ยนแปลงได้ทันที มีสีสันชัดเจน
⚠️ ข้อเสียมีกลุ่มมากเกินไปจะดูสับสน ควร ≤7 กลุ่ม
💧 แผนภูมิแบบน้ำตก (Waterfall chart)
แผนภูมิแบบน้ำตก (Waterfall chart) เป็นแผนภูมิแสดงผลรวมสะสมใช้ในการวิเคราะห์ค่าเริ่มต้นและเมื่อมีการเปลี่ยนแปลงค่าที่ได้จะเป็นเชิงบวก หรือเชิงลบ โดยคอลัมน์แรกและคอลัมน์สุดท้ายคือช่วงเวลาในการวิเคราะห์ข้อมูล

แผนภูมิแบบน้ำตกเหมาะสำหรับการแสดงข้อมูลที่มีค่าตั้งต้น การเปลี่ยนแปลงที่เกิดขึ้นในแต่ละช่วง และค่าสุดท้าย
💧
Waterfall Chart
แสดงการเปลี่ยนแปลงสะสม บวก/ลบ
Waterfall chart แสดงผลรวมสะสม โดยคอลัมน์สีเขียวคือค่าที่เพิ่มขึ้น สีแดงคือค่าที่ลดลง สีน้ำเงินคือค่าเริ่มต้นและสิ้นสุด ทำให้เห็นภาพรวมของการเปลี่ยนแปลงได้ชัดเจน
เมื่อไหร่ควรใช้: วิเคราะห์งบกำไรขาดทุน, การเปลี่ยนแปลงยอดขายสะสม, การวิเคราะห์ต้นทุนและรายได้
✅ ข้อดีเห็นการเปลี่ยนแปลงบวก/ลบได้ชัด แสดงผลสะสมได้ดี
⚠️ ข้อเสียซับซ้อนสำหรับผู้ที่ไม่คุ้นเคย ต้องอธิบายเพิ่มเติม
🔺 แผนภูมิกรวย (Funnel chart)
Funnel chart (กรวย) เป็นแผนภูมิแสดงค่าลำดับขั้นตอนต่างๆ ในกระบวนการเป็นกราฟแท่งแนวนอน ซึ่งเราสามารถเรียงลำดับและติดตามความคืบหน้าของข้อมูลแต่ละขั้นตอนได้

แผนภูมิกรวยเหมาะสำหรับข้อมูลที่มีลำดับ อาจเป็นขั้นตอนหรือกระบวนการทำงานต่างๆ โดยจะแสดงเป็นแท่งแนวนอน ซึ่งแท่งแรก (หรือลำดับแรก) จะชี้วัดค่าเป็น 100% ไล่เรียงลงไปจนถึงแท่งสุดท้าย เพื่อใช้ในการคาดคะเนโอกาสที่จะเกิดขึ้นในแต่ละเหตุการณ์ และดูความคืบหน้าในการทำงานแต่ละขั้นตอน
🔺
Funnel Chart
แสดงขั้นตอนกระบวนการ — ติดตามการเปลี่ยนแปลงแต่ละขั้น
Funnel chart แสดงข้อมูลแบบขั้นตอน แท่งแรกคือ 100% แล้วลดลงในแต่ละขั้น ทำให้เห็นว่าขั้นตอนไหนมีการ Drop ออกมากที่สุด
เมื่อไหร่ควรใช้: กระบวนการสมัครงาน, Sales Pipeline, ขั้นตอนการสมัครสินเชื่อ, จำนวนประชากรแต่ละช่วงอายุ
✅ ข้อดีเห็นการ Drop ในแต่ละขั้นตอนชัดเจน เปรียบเทียบ % ได้ทันที
⚠️ ข้อเสียต้องมีลำดับที่ชัดเจน ไม่เหมาะกับข้อมูลที่ไม่มีลำดับ
🔵 แผนภูมิกระจาย (Scatter chart)
Scatter chart (แผนภูมิกระจาย) เป็นแผนภูมิที่มีแกนค่าสองแกน (แกนค่าแนวนอน X และแกนค่าแนวตั้ง Y) ซึ่งจะแสดงข้อมูลเป็นชุด (ที่จุดตัดของค่าตัวเลข X และ Y) และสามารถตั้งค่าจำนวนของจุดข้อมูลได้สูงสุดถึง 10,000 จุดด้วยกัน

แผนภูมิกระจายเหมาะสำหรับการแสดงข้อมูลและการเปรียบเทียบค่าตัวเลข 2 ค่า ในกรณีที่ค่าตัวเลขในแกนนอนมีระยะห่างระหว่างค่าไม่สม่ำเสมอ หรือจุดข้อมูลในแกนนอนมีจำนวนมาก
🔵
Scatter Chart & Bubble Chart
หาความสัมพันธ์ระหว่าง 2 ตัวแปร
แสดงความสัมพันธ์ระหว่างตัวแปร 2 ตัว โดยแต่ละจุดแทนข้อมูล 1 รายการ ขนาดของ Bubble สามารถแสดงมิติที่ 3 ได้ รองรับสูงสุด 10,000 จุดข้อมูล
เมื่อไหร่ควรใช้: หาความสัมพันธ์อายุกับการเต้นของหัวใจ, ปริมาณจราจรกับรายได้, ค่าทางสถิติและข้อมูลเชิงลึก
✅ ข้อดีแสดง 2-3 มิติพร้อมกัน รองรับข้อมูลจำนวนมาก หา Outlier ได้ง่าย
⚠️ ข้อเสียอ่านยากหากจุดทับซ้อนกัน ต้องมีความรู้ทางสถิติ
🥧 แผนภูมิวงกลมและแผนภูมิโดนัท (Pie chart & Donut chart)
Pie chart & Donut chart เป็นแผนภูมิที่มีลักษณะคล้ายกัน (ต่างกันเพียงแผนภูมิโดนัทจะมีช่องว่างตรงกลาง) ใช้สำหรับแสดงความสัมพันธ์ของข้อมูล โดยแบ่งชุดข้อมูลตามสัดส่วนของผลรวมรายการทั้งหมด ซึ่งจะแสดงผลเป็นค่าเปอร์เซ็นต์

เราจะเลือกใช้แผนภูมิทั้ง 2 รูปแบบนี้ เมื่อเรามีชุดข้อมูลเพียง 1 ชุดเท่านั้น และไม่มีชุดข้อมูลใดมีค่าติดลบ และชุดข้อมูลในการสร้างแผนภูมินั้นไม่ควรแบ่งกลุ่มเกิน 7 กลุ่ม เพราะจะทำให้การแสดงผลอ่านค่าได้ยาก
🥧
Pie Chart (แผนภูมิวงกลม)
สัดส่วน % ของแต่ละกลุ่ม
แสดงสัดส่วนของแต่ละกลุ่มเทียบกับทั้งหมด เหมาะเมื่อมีหมวดหมู่ไม่เกิน 7 กลุ่ม และไม่มีค่าติดลบ
เมื่อไหร่ควรใช้: จำนวนผู้มีงานทำจำแนกตามภาค, ส่วนแบ่งตลาด, สัดส่วนงบประมาณ
✅ ข้อดีเข้าใจสัดส่วนได้ทันที สวยงามสำหรับ Presentation
⚠️ ข้อเสียมากกว่า 7 กลุ่มอ่านยาก ไม่เหมาะกับค่าใกล้เคียงกัน
🍩
Donut Chart (แผนภูมิโดนัท)
สัดส่วน + ค่าสรุปตรงกลาง
คล้าย Pie chart แต่มีช่องว่างตรงกลางสำหรับแสดงตัวเลขสรุปหรือ KPI สำคัญ ดูทันสมัยและสื่อสารได้มากขึ้น
เมื่อไหร่ควรใช้: แสดงสัดส่วนพร้อมยอดรวม, ส่วนแบ่งตลาดพร้อมมูลค่ารวม
✅ ข้อดีแสดงค่าสรุปตรงกลางได้ ดูโมเดิร์น
⚠️ ข้อเสียเหมือน Pie — ไม่ควรเกิน 7 กลุ่ม
🌳 แผนที่ต้นไม้ (Treemap)
แผนที่ต้นไม้ (Treemap) เป็นแผนภูมิที่แสดงข้อมูลแบบลำดับชั้นเป็นรูปสี่เหลี่ยมวางเรียงต่อๆ กัน โดยจะแบ่งลำดับชั้น 2 ระดับ คือ 'กิ่ง' จะแบ่งตามสีแต่ละกิ่ง Group และ 'ใบ' จะเป็นสี่เหลี่ยมย่อยเล็กๆ ภายในอีกทีจะแบ่งส่วนตามข้อมูลใน Value

เราจะใช้แผนภูมิต้นไม้สำหรับการเปรียบเทียบประเภทข้อมูลจำนวนมากที่มีลำดับชั้น โดยใช้รูปสี่เหลี่ยมพื้นสีแสดงข้อมูล ซึ่งจะแสดงขนาดตามสัดส่วนข้อมูลภายในลำดับชั้นนั้นๆ
🌳
Treemap (แผนที่ต้นไม้)
ข้อมูลลำดับชั้น 2 ระดับ: กิ่ง (Group) และ ใบ (Value)
Treemap แสดงข้อมูลลำดับชั้นเป็นสี่เหลี่ยม ขนาดใหญ่ = ค่ามาก สีเดียวกัน = กลุ่มเดียวกัน เหมาะกับข้อมูลที่มีหมวดหมู่จำนวนมาก
เมื่อไหร่ควรใช้: ยอดขายแยกตามหมวดสินค้าและประเภทย่อย, งบประมาณแยกแผนกและโครงการ, Export by country
✅ ข้อดีใช้พื้นที่ได้เต็มประสิทธิภาพ รองรับข้อมูลจำนวนมาก เห็นสัดส่วนชัด
⚠️ ข้อเสียกล่องเล็กมากอ่านชื่อยาก ไม่เหมาะกับค่าที่ใกล้เคียงกันมาก
🗺️ แผนที่และแผนที่แถบสี (Map และ Filled Map)
Map (แผนที่) และ Filled Map (แผนที่แถบสี) เป็นแผนภูมิรูปภาพที่แสดงแผนที่ทั่วโลกโดยอัตโนมัติ ใช้เปรียบเทียบค่าและแสดงประเภทข้อมูลต่างๆ ซึ่งทั้ง 2 แบบนั้นจะมีลักษณะที่แตกต่างกัน ดังนี้

เราจะใช้แผนที่เมื่อต้องการเปรียบเทียบค่าทางภูมิศาสตร์ เช่น จำนวนประชากรทั้งประเทศ หรือค่าตัวเลขที่แสดงข้อมูลในแต่ละพื้นที่
📍
Map (แผนที่จุด Bubble)
แสดงค่าเป็นขนาดวงกลมบนแผนที่
แสดงข้อมูลบนแผนที่ในรูปแบบวงกลม (Bubble) ขนาดวงกลมสื่อถึงปริมาณข้อมูล เหมาะกับข้อมูลที่มีพิกัด GPS หรือชื่อเมือง/จังหวัด
เมื่อไหร่ควรใช้: ปริมาณจราจรแต่ละจังหวัด, จำนวนสาขาแต่ละพื้นที่, ยอดขายแยกตามตำแหน่ง
✅ ข้อดีเห็นการกระจายตัวเชิงพื้นที่ได้ทันที
⚠️ ข้อเสียต้องการ Bing Map License, ข้อมูลชื่อสถานที่ต้องชัดเจน
🗺️
Filled Map (แผนที่แถบสี)
เน้นสีพื้นที่ตามค่าข้อมูล
ระบายสีพื้นที่ตามค่าข้อมูล สีเข้มกว่า = ค่ามากกว่า เหมาะกับข้อมูลที่แบ่งตามเขตภูมิศาสตร์ เช่น จังหวัด ภูมิภาค ประเทศ
เมื่อไหร่ควรใช้: จำนวนประชากรแต่ละจังหวัด, อัตราการว่างงานแต่ละภาค, ยอดขายแยกประเทศ
✅ ข้อดีเห็นภาพรวมทางภูมิศาสตร์ได้ชัด ดูเป็นมืออาชีพ
⚠️ ข้อเสียต้องการชื่อพื้นที่ถูกต้องตาม Bing Map
🔢 การ์ด, KPI และตัวแบ่งส่วนข้อมูล
Visual กลุ่มนี้ใช้สำหรับแสดงตัวเลขสำคัญ ตัวชี้วัดหลัก และการควบคุม Filter ใน Dashboard
💳
การ์ด (Card)
Single Value — แสดงตัวเลขเดียวที่สำคัญ
แสดงตัวเลขเดียวที่สำคัญที่สุด เช่น ยอดขายรวม กำไรสุทธิ จำนวนลูกค้า เหมาะสำหรับวางเป็น Highlight บน Dashboard
เมื่อไหร่ควรใช้: KPI ที่ต้องการเน้น, ตัวเลขหลักของ Dashboard
✅ ข้อดีเข้าใจทันที เน้นตัวเลขสำคัญ
⚠️ ข้อเสียไม่แสดงบริบทหรือการเปลี่ยนแปลง
🎯
KPI Visual
เปรียบเทียบค่าจริง vs เป้าหมาย
แสดงค่าจริง เปรียบเทียบกับเป้าหมาย พร้อม % ความแตกต่าง และ Trend แสดงแนวโน้ม ใช้สำหรับการเปรียบเทียบยอดขายปีปัจจุบันกับปีที่ผ่านมา
เมื่อไหร่ควรใช้: ติดตามผลการดำเนินงาน vs เป้าหมาย, ยอดขายปีนี้ vs ปีที่แล้ว
✅ ข้อดีเห็นทั้งค่าจริง เป้าหมาย และแนวโน้มในที่เดียว
⚠️ ข้อเสียต้องการข้อมูล 2 ตัว (จริง + เป้า)

ตัวแบ่งส่วนข้อมูล (Slicer)
Filter แบบ Interactive
Slicer เป็น Visual สำหรับกรองข้อมูล (Filter) แบบ Interactive ให้ผู้ใช้สามารถเลือก กดปุ่ม หรือเลื่อน Slider เพื่อกรองข้อมูลในทุก Visual บน Dashboard
เมื่อไหร่ควรใช้: ทุก Dashboard ควรมี Slicer สำหรับกรองตาม วันที่, ปี, ภูมิภาค, หน่วยงาน
✅ ข้อดีผู้ใช้ควบคุม Filter ได้เอง ไม่ต้องแก้ไข Report
⚠️ ข้อเสียหลาย Slicer อาจทำให้ Dashboard ดูรกและช้า

การ์ดหลายแถว (Multi-row Card)
แสดง KPI หลายตัวในกล่องเดียว
แสดง KPI หลายรายการพร้อมกันในพื้นที่น้อย เหมาะสำหรับ Summary Dashboard ที่ต้องแสดงตัวชี้วัดหลักหลายตัว
เมื่อไหร่ควรใช้: Executive Dashboard, สรุปผลการดำเนินงานประจำเดือน
✅ ข้อดีประหยัดพื้นที่ ดูหลายค่าพร้อมกัน
⚠️ ข้อเสียไม่สามารถแสดงกราฟร่วมได้

Visual กลุ่มนี้ใช้สำหรับการวิเคราะห์ข้อมูลเชิงลึก การสร้างสรุปอัตโนมัติ และการแสดงข้อมูลในรูปแบบตาราง

R Script Visual
สคริปต์รูปภาพ R
ใช้โค้ด R ในการสร้างกราฟและการวิเคราะห์ทางสถิติขั้นสูง รองรับ Library เช่น ggplot2, forecast, plotly
เมื่อไหร่ควรใช้: ใช้ข้อมูลในอดีตที่ผ่านมาเพื่อนำมาวิเคราะห์เหตุการณ์ที่จะเกิดในอนาคต, Statistical Analysis
✅ ข้อดีทำกราฟซับซ้อนได้ รองรับ Statistical Model
⚠️ ข้อเสียต้องมีความรู้ภาษา R ติดตั้ง R บนเครื่องก่อน
Py
Python Visual
ภาพ Python
ใช้โค้ด Python ในการสร้าง Visualization และ Machine Learning โดยใช้ Library เช่น matplotlib, seaborn, scikit-learn
เมื่อไหร่ควรใช้: Data Science, Machine Learning, วิเคราะห์ข้อมูลเชิงลึกด้วย AI
✅ ข้อดีรองรับ ML/AI ทำ Forecast และ Clustering ได้
⚠️ ข้อเสียต้องติดตั้ง Python และ Package ก่อนใช้งาน

Smart Narrative Visual
สรุปคำบรรยายอัตโนมัติ
สร้างคำบรรยายแบบสรุปข้อมูลวิชวล โดยระบุเป็นประเด็นสำคัญ เช่น "ยอดขายสูงสุดในเดือนมีนาคม เพิ่มขึ้น 23% จากเดือนก่อน"
เมื่อไหร่ควรใช้: สร้างคำบรรยายแบบสรุปข้อมูลวิชวล โดยระบุเป็นประเด็นสำคัญ
✅ ข้อดีสรุปข้อมูลอัตโนมัติ ลดเวลาเขียนรายงาน
⚠️ ข้อเสียภาษาไทยยังไม่สมบูรณ์ ต้องตรวจสอบความถูกต้อง

ตาราง (Table) และแมทริกซ์ (Matrix)
แสดงข้อมูลดิบในรูปแบบตาราง
Table แสดงข้อมูลดิบแบบตาราง เหมาะสำหรับดูรายละเอียด | Matrix คล้าย PivotTable ใน Excel รองรับการ Drill Down ตามลำดับชั้น
เมื่อไหร่ควรใช้: รายงานที่ต้องการดูข้อมูลรายละเอียด, สรุปข้อมูลหลายมิติ
✅ ข้อดีแสดงข้อมูลครบถ้วน รองรับ Sort, Filter, Drill Down
⚠️ ข้อเสียดูยากเมื่อข้อมูลมาก ไม่ใช่ Visual ที่ดึงดูดสายตา
